━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Old signup only created Firebase Auth user, NOT Firestore doc

  ✅ Auto-recovery - login verifies and creates missing docs

Result:
  User signs up → Document appears in Firestore immediately
  Admin dashboard shows user right away

Root Cause:
  Images uploaded but URLs not linked to item

  Or images not compressed (slow, large)

Solution:
  ✅ storageUploadItemImages() - Upload + compress + get URL
  ✅ Automatic compression via Canvas API (50-70% size reduction)
  ✅ Thumbnail generation for admin display
  ✅ Proper URL linking in Firestore doc
  ✅ Storage rules allowing authenticated uploads

Result:
  User uploads image → Compressed → Uploaded → URL saved to Firestore
  Complete flow in one function call
